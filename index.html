    Database Schema

    📋 SUPPLIERS
    • 🔑 id VARCHAR (PK)
    • name VARCHAR
    • contact VARCHAR
    • email VARCHAR
    • address TEXT
    • status ENUM(Active/Inactive)
    📦 PRODUCTS
    • 🔑 id VARCHAR (PK)
    • name VARCHAR
    • category VARCHAR
    • price DECIMAL
    • 🔗 supplierId FK → Suppliers
    • stock INT
    📑 ORDERS
    • 🔑 id VARCHAR (PK)
    • 🔗 supplierId FK → Suppliers
    • 🔗 productId FK → Products
    • quantity INT
    • orderDate DATE
    • status ENUM(Pending/Delivered)
    💳 PAYMENTS
    • 🔑 id VARCHAR (PK)
    • 🔗 orderId FK → Orders
    • amount DECIMAL
    • paymentDate DATE
    • status ENUM(Paid/Unpaid)
    🔗 Relationships
    SUPPLIERS ──1:N──▶ PRODUCTS ──1:N──▶ ORDERS ──1:1──▶ PAYMENTS

    One Supplier → Many Products | One Product → Many Orders | One Order → One Payment
